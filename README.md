**SusList** — реестр цифровых активов, используемых в схемах с высокими рисками и признаками недобросовестности. База формируется на основе анализа мониторинга рыночной активности.

Основная цель проекта — предоставление актуальных индикаторов компрометации для автоматической блокировки на уровне почтовых шлюзов, DNS-фильтров, АТС и CRM-систем с целью предотвращения B2B-мошенничества, фишинга и хищения ТМЦ.

### 🧩 Формат данных и Схема NDJSON (`suslist.ndjson`)

Файл представляет собой NDJSON-фид (Newline Delimited JSON), где каждая строка является независимым JSON-объектом.
Этот формат оптимизирован для потоковой обработки: скриптам не нужно загружать весь файл в оперативную память, а добавление/удаление записей не требует переписывания всего массива.

**Пример записи:**
```json

{"domain": "exemplegroup.ru", "match_mode": "subtree", "email": null, "phone": null, "target_id": {"type": "inn", "value": "0000000000"}, "target_name": "ООО «ПРИМЕР»", "first_seen": "2024-12-22", "last_updated": "2026-06-04", "source_reports": ["RPT-04-20260529"], "note": "Добавление суффикса 'group'."}
```
**Описание полей:**

| Поле | Тип | Описание |
| :--- | :--- | :--- |
| domain| string\null | Доменное имя.|
| match_mode | string\null | Режим блокировки: subtree (блокирует домен и все поддомены), exact (точное совпадение), wildcard (только поддомены). Для записей только с телефоном — null.|
| email | array|null | Массив email-адресов, отличных от выявленного домена. null, если не выявлено.|
| phone | array|null | Массив телефонных номеров в формате E.164 (напр., +79876543210), null, если не выявлено.|
| target_id | object\null | Идентификатор имитируемой организации: {type: inn\ogrn\reg_code, value: ...}. null, если организация не установлена.|
| target_name | string\null | Наименование имитируемой организации (для справки). null, если организация не установлена.|
| first_seen | date | Дата первого обнаружения (YYYY-MM-DD).|
| last_updated | date | Дата последнего обновления записи (YYYY-MM-DD).|
| source_reports | array | Массив источников (напр., RPT-04-20260529).|
| note | string | Дополнительный аналитический комментарий.|

### 📄 Sus Domain List (`SDL.txt`)

Содержит только домены.
```txt
# SDL.txt — Sus Domain List
# Updated: 2026-06-04
# Format: apex domains only.
# Total entries: 4
exemplegroup.ru
groupexemple.ru
exemplegrp.ru
exemplegr.ru
```
### 📄 Sus Mail List (`SML.txt`)

Содержит только почтовые адреса.
```txt
# SML.txt — Sus Mail List
# Updated: 2026-06-04
# Format: email only.
# Total entries: 4
info@exemplegroup.ru
user@groupexemple.ru
*@exemplegrp.ru
office@exemplegr.ru
```

### 📄 Sus Phones List (`SPHL.txt`)

Содержит только телефоны в строгом формате E.164 (без пробелов, скобок и дефисов), готовом для импорта в любую АТС.
```txt
# SPHL.txt — Sus Phones List
# Updated: 2026-06-04
# Format: E.164 strictly (e.g., +79991234567). No spaces, brackets, or dashes.
# Total entries: 4
+79876543210
+79876543201
+79876543021
+79876540321
```
